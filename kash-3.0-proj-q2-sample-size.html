Projects §9 · Q2 — When are duration estimates trustworthy?

Each card shows the estimate for the same task ("write docs") at n=1, n=3, n=8 past samples. The fork is how each model treats thin data.

A · Silent until N (≈3)

Withhold until trustworthy

n=1— no estimatelearning…
n=3~4hshown
n=8~3.5hshown
Nothing appears until the sample crosses the threshold.
  • Never misleads with thin data
  • Simple rule
  • No help in the early days when you’d value it
B · Always show + confidence label

Transparent from sample one

n=1~3hlow · 1
n=3~4hfair · 3
n=8~3.5hgood · 8
Estimate always shown, tagged with confidence + sample count so you weight it yourself.
  • Useful immediately; honest about uncertainty
  • Teaches you to read confidence
  • A low-confidence number can still anchor
C · Graduated tiers

“Rough” → “estimate” → “confident”

n=1rough: ~3hrough
n=3est: ~4hestimate
n=8~3.5hconfident
Same as B but confidence is expressed in words/styling rather than a count.
  • Plain-language; gentle on numbers
  • Fuzzier than an explicit n; tiers to define
D · Always a range, narrowing

No false-precision single number

n=12–6h
n=33–5h
n=8~3.5h
Always a band that tightens with data; collapses to a point only when confident.
  • Visually honest; resists false precision
  • Useful from n=1
  • Ranges are harder to schedule/sum against

Note: the threshold N (≈3 shown here) is a tunable default, not the fork — the fork is which presentation above. We can set/confirm N once the model is chosen.
