Today §6 · Q2 — Balance bar weighting (same day, four models)

The point of the bar is to reveal a lopsided day. Watch how the same tasks produce very different “balance” depending on what we count.

Example day — ★ = Top-3 (deep work), plain = small task. Durations = actual focus-block time the Calendar records.
Pro: ship deck  90m Pro: 2 emails  12m Proj: prototype  45m Adult: renew reg  8m Adult: pay bill  6m Adult: schedule dentist  5m Body: walk  18m
A · Pure count

Every task = 1

Counts: Pro 2 · Proj 1 · Adult 3 · Body 1  →  errands look like the biggest part of the day.

Simplest, but a pile of 5-minute errands can outweigh a 90-min deliverable. Misleads on an ADHD day where small tasks multiply.

B · Top-3 weighted · matches §2 v1 ✓

Top-3 = 3 units, small = 1

Weighted: Pro 4 · Proj 3 · Adult 3 · Body 1  →  deep work leads; errands shrink to their real weight.

Same metric as §2, so Today and Categories agree. No new data needed — uses the Top-3 flag you already set. Upgrade path to time-based stays open.

C · Time-based (actual minutes)

Use recorded focus durations

Minutes: Pro 102 · Proj 45 · Adult 19 · Body 18  →  truest picture of where the day actually went.

Today uniquely has this data (the Calendar records durations). But it’s a record, not a plan — early in the day the bar is near-empty, and untimed tasks don’t register. §2 defers time-based balance globally.

D · Hybrid (weighted now → time later)

Ship B, designed to swap to C

Renders as B today; same component swaps its input to minutes once time-tracking is trusted (the §2 “upgrade later” path).

Lowest-risk: ships now with no new data, keeps the door open to C without reworking the view. Slightly more build scaffolding.
